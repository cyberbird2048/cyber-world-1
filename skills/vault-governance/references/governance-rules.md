# Vault Governance 治理规则

## 豁免目录（不上报孤立笔记）

以下目录的笔记本来就是独立文档，不需要被其他笔记链接到：

| 目录 | 原因 |
|------|------|
| `09_System/claude-sync/` | Claude Projects 自动同步的技术文档，是原始素材 |
| `09_System/Skills/` | Skill 知识索引卡，独立存在 |
| `09_System/Automation/` | 系统自动生成的日报/报告，非知识内容 |
| `01_Projects/` | 项目文件，结构化产出，不需要引用 |
| `systems-skill/` | 外部导入的系统 skill，只读 |
| `openclaw_migration/` | 历史迁移数据 |

## 核心内容目录（孤立笔记高置信）

以下目录的笔记**应该**被其他笔记链接到，孤立意味着知识孤岛：

| 目录 | 期望 |
|------|------|
| `07_Wiki/` | Wiki 条目应被多笔记引用 |
| `05_thought/` | 思考文章应有背景链接 |
| `03_Resources/` | 参考资料应被项目引用 |
| `06_me/` | 个人记录应有连接 |

## 断链假阳性模式

以下 wikilink 目标不算真正断链：

1. **模板占位符**：`[[相关笔记1]]`、`[[AreaName]]`、`[[NoteName]]` 等
2. **代码示例**：link 目标含 `{`、`}`、`"`、`=`、`$` 等特殊字符
3. **技术术语**：全大写 (`API`、`JSON`、`CRUD` 等，不是笔记名)
4. **来自 claude-sync**：`09_System/claude-sync/` 里的断链多为 skill 示例代码

## 空白笔记豁免

以下情况不算问题：

- `00_Inbox/` 里的 stub（待处理收件箱）
- 文件名含 `README`、`CLAUDE`、`AGENTS`、`_registry` 的系统文件
- `09_System/` 下的任何笔记

## 修复优先级矩阵

| 类型 | 优先级 | 推荐动作 |
|------|--------|----------|
| 过期笔记 | P0 | 立即处理：删除/归档/更新日期 |
| 07_Wiki 孤立笔记 | P1 | 在相关主题中补链接 |
| 核心内容区断链 | P1 | 创建目标笔记或修正链接 |
| 07_Wiki 空白笔记 | P2 | 补充内容或删除 |
| 其他孤立笔记 | P3 | 定期整理，不紧急 |
| 系统目录问题 | 忽略 | 自动豁免，不上报 |

## 健康度定义

| 健康度 | 高置信问题数 | 含义 |
|--------|-------------|------|
| 🟢 健康 | 0-3 | vault 连接密度良好 |
| 🟡 需关注 | 4-15 | 有知识孤岛，建议定期整理 |
| 🔴 需处理 | 16+ | 大量断链/孤岛，知识图谱质量下降 |
