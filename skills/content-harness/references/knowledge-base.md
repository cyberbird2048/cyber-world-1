## 高效模式区

- [opinion] AI 视角叙事：以观察者身份说话，关键是"我看见了什么"而非"我有多少数据"——身份证不应亮给读者，洞察才是入场券
- [opinion] 核心类比必须贯穿到结尾：开头引入"和不存在的人赛跑"的类比，结尾"停止追一个不存在的人，是真正开始跑的第一步"完成闭环
- [opinion] 反直觉升级：不只是第一次提出反直觉，要在中段加一层深化——"追得越用力，他跑得越快"把类比推进了一级
- [all] 去掉 AI 身份证亮牌（"我见过的数据比你多，我可以保证"）——听起来像情感客服报告，不像真人说话
- [all] 具体可做的事：在"分析焦虑原因"段落后加一句具体行动（"有一件具体的事你可以做：停止追那个虚构的自己"），把共鸣转化为方向

- [news-reaction] 开头判断必须可被质疑，有具体所指，不是情绪性的悬念修辞
- [news-reaction] 技术论据必须配最小可感知的场景例子，否则读者无法判断真实性
- [news-reaction] 结尾给作者自己的行动/立场，比"让读者自己决定"更有力
- [all] 反向否定升华句（"没有听起来那么小"）是 AI 高频收尾特征，用具体行动替代

- [opinion] 核心类比贯穿全文，凝聚力最强
- [opinion] 场景开头，金句结尾收束——V型弧完整
- [opinion] 补强论证时用具体人物/案例，不用纯说理
- "你以为A了"替代"不是A而是B"——反直觉更强且不触发规则
- 单句成段制造节奏感
- [creator-share] 开头直接展示作品+核心数字，过程叙述轻盈，洞察前置
- [creator-share] 过程窗口需要有具体创作决策（为什么分5层），不能只有动机（朋友来问）

## 避免模式区

- 禁止词：随着…的发展、值得注意的是、总的来说、不可否认、在当今…时代、毋庸置疑
- "不是A而是B"全文不超过2次（首次运行发现AI高频特征）
- 金句不能用于开局——读者没有经历过弧线就没有共鸣
- 判断力/认知框架/思维方式——三个概念必须桥接，不能跳跃
- [creator-share] 配套资源（H5/网站）存在时，文章不要复述全部内容——展开1-2个感触最深的点，其余引导去看原文
- [creator-share] 洞察前置到全文30%处，不能藏在最后当附赠品
- [creator-share] 中段连续信息罗列（5层逐一展开+路线图+费用）= 产品说明书，丢失对话感
- [opinion] 水段特征：删掉后文章不变差。常见水段：分析性解释段（"这是真实不确定性和信息超载叠加的结果"）、重复的情感升华段。直接切，不解释
- [发布流程] 微信封面/插图必须通过 filetransfer API 上传 + operate_appmsg API 绑定，禁止用 CDP eval 操作编辑器 UI 上传文件
- [发布流程] operate_appmsg 更新草稿时 content0 字段必须包含完整 HTML，缺失则正文被清空
