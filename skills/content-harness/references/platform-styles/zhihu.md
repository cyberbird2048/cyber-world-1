# 知乎发布风格指南

> 从知乎高赞内容规律中提炼的发布策略。
> Stage 9 知乎发布流程的核心参考文档。

---

## 平台特性

- **核心内容形态：** 回答（问题回答） / 文章（专栏）
- **两种入口，策略不同：**
  - **问题回答**：搜索关键词后选热门问题作答，自然流量最高
  - **专栏文章**：类似公众号，靠粉丝和推荐流分发
- **读者画像：** 知识密度要求高、对"证据"和"逻辑"敏感、对自我包装反感
- **算法偏好：** 赞同数 > 收藏 > 评论；专业感 > 情感共鸣（与小红书相反）

## 内容风格规范

### 文章类型路由

| 公众号文章类型 | 知乎适配形式 | 入口选择 |
|------------|-----------|---------|
| opinion | 专栏文章 或 问题回答 | 找对应的热门问题作答 |
| creator-share | 问题回答（"用了X工具是什么体验"类问题） | 找工具相关问题 |
| how-to | 问题回答（"怎么用X做Y"类问题） | 找操作方法类问题 |
| news-reaction | 专栏文章 | 搭热点 |
| life-reflection | 专栏文章 | 较少用，仅在情感共鸣强时 |

### 标题格式（专栏文章）

```
[判断 / 反常识 / 数字] + [具体场景]
```

有效格式：
- **判断式**：「AI写作工具正在分化：这类创作者会被替代，那类会被放大」
- **反常识**：「我用 AI 写了3个月内容，发现它最厉害的地方不是速度」
- **问题式**（适合回答型）：「普通人用 AI 写公众号，能写出什么质量？」

**禁止：** 标题党（内容不支撑标题）、夸张词（"颠覆""震惊"）

### 开头规范（决定是否被折叠阅读）

知乎读者判断是否值得读的关键：**开头 100 字内必须给出核心判断或有效信息**。

有效开头：
```
直接给出核心结论，然后展开。
"3个月用下来，结论是：AI 写作工具对内容创作者的价值在速度，不在质量。原因如下。"
```

无效开头：
```
背景铺垫型："随着AI技术的发展，越来越多的人开始..."（知乎读者会直接跳过）
```

### 正文结构（专栏文章）

```
核心判断（1-2段，100字内）
↓
论据展开（3-5个要点，每个配具体案例/数据）
↓
反驳可能的质疑（1-2段，增加可信度）
↓
结论 + 实践建议（1段）
```

**字数：** 1000-2500 字最佳；超过 3000 字折叠风险高

**格式规范：**
- 可用 H2/H3 副标题（与公众号不同，知乎支持且读者习惯）
- 引用数据必须标注来源（"据 X 报告"，不需要链接但要有名称）
- 列表用 `-` 或 `1. 2. 3.`（知乎 Markdown 支持）

### 问题回答结构

```
结论先行（1句话）
↓
个人经历/案例（真实感，让读者信任）
↓
核心方法/观点（2-4条，带具体操作）
↓
一句话收尾（不是"欢迎关注"类的广告语，是真实的个人立场）
```

## 知乎 vs 公众号内容差异对照

| 维度 | 公众号 | 知乎 |
|------|-------|-----|
| 开头 | 场景/情绪切入 | 结论/判断先行 |
| 语气 | 口语化，感性 | 理性，有专业感 |
| 结构 | 无副标题，用空白行分节 | 可用副标题，结构更清晰 |
| 图片 | 多图（封面+插图） | 少图或无图（文字为主） |
| 结尾 | 金句/邀请/祝福 | 实践建议/个人立场 |
| 互动 | 转发分享 | 点赞收藏评论 |

## Stage 9 知乎发布流程

### 前置条件

- 公众号版文章已完成
- 用户在 Stage 7 确认了"发布到知乎"
- 已有 Stage 6 生成的平台版本草稿

### Step ZH-1: 确定知乎入口

分析文章类型，决定是"专栏发文"还是"问题作答"：

```
1. 如果文章是 news-reaction 或 opinion（时效性强）→ 专栏文章
2. 如果文章是 how-to 或 creator-share → 搜索知乎相关问题
   - 搜索关键词：文章标题的核心词 + "怎么" / "体验" / "推荐"
   - 选择：浏览量 10万+ 且近期有新回答的问题（活跃度保证）
   - 如果找不到合适问题 → 退回专栏文章
```

### Step ZH-2: 内容转化

**专栏文章路径：**
1. 标题改写：口语化 → 判断性/分析性（参考上方格式规范）
2. 开头改写：场景切入 → 结论先行（100字内给出核心判断）
3. 结构调整：添加副标题，重组为"判断→论据→反驳→结论"
4. 语气微调：删除强情绪词，改为更理性的表达（"有意思的是" 替换 "靠，真的假的"）
5. 结尾改写：删除感性收束，改为"基于以上，我的建议是..."

**问题回答路径：**
1. 压缩到 800-1500 字（回答不需要太长）
2. 第一段直接回答问题（"用过，结论是..."）
3. 提炼 3-4 个最有价值的要点，每点配 1 个具体例子
4. 结尾不推广，留个人判断

### Step ZH-3: 发布（手动）

**知乎暂无可靠 API 发布通道。** 流程输出可直接粘贴的内容：

```
--- 知乎发布内容包 ---
【发布类型】专栏文章 / 问题作答

【如果是问题作答，问题链接】
[知乎问题URL或描述]

【标题（专栏）/ 首句（回答）】
[内容]

【正文（已格式化为知乎 Markdown）】
[正文内容]
```

展示给用户，用户手动在知乎创作中心发布。

### Step ZH-4: 效果回收（发布后 48 小时）

提醒用户记录：赞同数、收藏数、评论数、阅读数。
记录到 knowledge-base.md 的知乎效果区域，供下次发布参考。
