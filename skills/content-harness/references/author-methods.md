# 参考作者写作方法论

> 从标杆文章和公开内容中提取的写作手法。写作时作为风格注入的参考。

## 数字生命卡兹克

> 张仁杰，95后，前AI业务总监。公众号10个月100+篇原创，AI领域影响力排名第二。
> 自定位"AI殿堂的门童"——不是技术布道者，而是把AI世界的好东西带给更多人看。
> 已开源个人写作skill（GitHub: KKKKhazix/Khazix-Skills），是极少数公开自己写作方法论的创作者。

### 核心风格：情感驱动 + 体验深度 + 哲学升维

卡兹克的文章像一个技术很强但极感性的朋友深夜跟你聊天。不以技术深度见长，而以**体验深度**和**应用想象力**见长。

### 开头技法

**从不用空洞概念开头**，每篇都从一个具体的、可感知的场景或事件启动：

1. **个人崩溃时刻**：从巴塞罗那出差时"我靠，我怎么什么都不记得了"切入（《写日记》）
2. **反直觉声明**："我知道这个观点可能有些暴论，从一个AI博主嘴里说出来会非常奇怪"（《劝你先别急着用AI》）
3. **具象数据冲击**："做了AI博主后，每天平均有八百个问题问我"（《100篇文章喂给AI》）
4. **事件即时感**：直接从新闻事件切入，营造"刚刚发生"的紧迫感（《DeepSeek一夜之间》）

### 标题公式

| 模式 | 实例 |
|------|------|
| 反常识/悖论 | 《作为一个AI博主，我劝你先别急着用AI》 |
| 概念造词 | 《DeepSeek的极致谄媚，正在摧毁我们的判断力》 |
| 叙事悬念 | 《当我把我的100篇文章喂给AI——坏了，我成数字生命了？》 |
| 主体+大判断 | 《腾讯把我的公众号，变成了一个巨大的"数字生命"》 |

**共性：** 口语化、有态度、有画面感。绝不用"深度解析""全面盘点"这类媒体八股词。

### 正文技法

1. **权威锚点**：引用具体人物的具体观点（马伯庸、罗振宇），不是泛泛引用"有人说"
2. **语言极端化**：用"我靠"、"爽爆了"、"废物仙人掌"这类情绪饱满的词。不怕粗，怕平
3. **游戏化叙事**：把技术探索写成游戏闯关——"白板→绿色→蓝色→紫色→金色传说"
4. **先体验后分析**：《Sora 2实测》不从技术原理讲起，从"八大场景"实际使用出发
5. **用具象数据替代模糊描述**："693张MidJourney图、185个Gen2镜头"而非"大量素材"
6. **碎片式表达**：大量短句、碎句、不完整的句子。"别这样。"单独成段
7. **造词能力**："搜索退化症""AI殿堂的门童"——一个新造的词概括复杂现象
8. **真实情绪外露**：愿意公开改变立场，"之前说AI不进成片，在看到这次发布后收回这句话"

### 结尾技法

**哲学升维 + 诗意收束**：

- 从日记技巧升维到"你这辈子真正活过的证据"
- 从 CLI 宠物升维到"跟概率较劲是人类最古老的浪漫"

**特征：** 最后 3-5 个短段落节奏像诗。以**态度/立场/展望**收尾，而非总结摘要。

### 四层自检体系（卡兹克自己定义的，极有参考价值）

1. **硬性规则层**：基础格式和禁止项
2. **风格一致性**：开头是否从具体场景切入？节奏是否有长短句交替？口语化是否充分？
3. **内容质量**：每个观点是否有具体场景支撑？知识是否以聊天感呈现？
4. **活人感终审**（最关键）：整篇文章读下来，是否像"一个有见识的普通人在认真跟你聊天"，而非"AI在输出信息"

### 卡兹克的写作迭代方法论（可复制到我们的系统）

1. 输入代表性文章 + 方法论白皮书
2. 让AI根据方法论生成初稿
3. 人工亲自改稿
4. 将AI版本和人工版本**同时输入AI，分析差异**，迭代回方法论

→ 这个"人机对比迭代"是最有价值的可复制模式。我们的 Stage 8 沉淀学习就是做这件事。

### 禁忌

- 不写"随着 AI 的发展…"类开头
- 不做居高临下的说教
- 不在技术细节前面道歉（"这部分可能有点枯燥"）
- 不用"笔者""作者"，全程"我"
- 不用"综上所述""深度解析"等八股词

## 花叔

> 陈云飞，网名 Alchain / AI进化论-花生，全网 30万+ 粉丝。独立开发者，代表作「小猫补光灯」AppStore 付费榜 Top1。
> 96+ 篇原创，覆盖 AI 工具实测、独立开发心得、产品思维方法论。
> 开源了「女娲.skill」「X导师.skill」，践行"内容即产品"——把方法论做成可运行的工具。

### 核心风格：实践先行 + 框架思维 + 犀利判断

花叔的核心原则是**"先做再说"**——先自己动手做产品/跑数据，再把过程和结论输出。他像一个在你旁边给你演示的朋友，不是居高临下的专家。

### 开头技法

1. **反差冲击**：制造"不可能→可能"的认知反差
   - "0基础" + "年入百万"
   - "不会写代码" + "AppStore 付费榜第一"
   - "1小时" + "做出一款 App"

2. **戏剧性事件 + 概念跳跃**：
   - 《女娲》：从"同事.skill"这个梗 → 跳跃到"为什么只蒸馏同事？你可以蒸馏芒格"
   - 《疯狂一周》：OpenClaw 封杀 → 4 天后 Anthropic 自己做了一个

3. **故事化切入**：从一个具体的人/事件开始，不直接讲结论
   - "2020年12月，Dario Amodei 从 OpenAI 辞职。他不是一个人走的，14个研究员跟着他一起离开..."

4. **首发/独家标签**：抢时效性制高点
   - "首发解读 51 万行代码！"
   - "【首发】Google 发布 Gemini 3"

### 正文技法

1. **原创框架**：不引用别人的模型，自己提。"行为蒸馏 vs 认知蒸馏"、"Agent OS 三层架构"
2. **表格化对比**：复杂概念用表格一目了然
3. **毒舌类比**：
   - "办了一张健身房年卡，然后每天带 18 个朋友去洗澡"（API 滥用）
   - "先请你吃饭，再卖你保险"（平台策略）
4. **诚实边界**：明确说"这个系统做不到 X"。承认局限反而建立信任
5. **多事件串联**：把多个看似独立的事件串成一条叙事线
6. **洋葱模型分层**：同一内容服务不同层次读者
   - 外层：生活化比喻和个人故事（吸引非技术用户）
   - 应用层：具体操作步骤和工具使用
   - 原理层：技术本质解释
   - 方法论层：可复制的思维框架

### 结构模式

| 模式 | 公式 | 适用场景 |
|------|------|---------|
| 反差叙事 | [低起点] + [高成就] + [具体路径] | 个人经验、案例拆解 |
| 首发解读 | [热点事件] + [独家角度] + [深度分析] | 新产品/行业事件 |
| 数字锚定教程 | [N个方法] + [工具名] + [效果承诺] | 教程、实操指南 |
| 横向对比 | [A vs B vs C] + [多维度测试] + [场景化推荐] | 工具对比、选型 |
| 故事线分析 | [人物故事线] + [商业/技术洞察] | 人物报道、行业分析 |

### 结尾技法

**一句话重新定义全文**：

- "女娲造的不是人。是镜子。"
- "围墙花园最怕的，是隔壁有人在建一个没有墙的花园。"

**特征：** 极短，通常一句话。用对仗或反转结构。能单独发朋友圈。

### 信任链构建（花叔独有的方法论，值得学习）

**亲自做 → 做出成绩 → 分享方法 → 证明可复制**

他不是"教你做但自己没做过"的人，AppStore 排名就是最硬的信任背书。他的反直觉建议因此有说服力："先做十个垃圾产品，不要追求完美"。

### 禁忌

- 不写没有原创框架的纯转述（"Anthropic 发布了 X, Y, Z"）
- 不做没有立场的中性评价（"各有优劣"——必须有判断）
- 不用"让我们拭目以待"这种不表态的结尾
- 不做没有实践支撑的纯理论分析

## 刘小排

### 核心风格：实战数据 + 反差身份 + SOP 清单体

刘小排的内容像一个"代码写得很烂但年入千万"的朋友在教你赚钱。不讲原理，只讲"怎么用"和"用了会怎样"。

### 开头技法

**极端数据冲击**：用两个数字的巨大落差制造点击冲动。

- "$200 月费 vs $50,000 实际消耗"
- "数周建模 vs 8 秒完成"
- "一个人 vs 以前几十个人的团队"

**否定式共鸣**：直接说出读者心中的痛点。

- "AI 一开始写得很顺，越改越乱，最后变成屎山。"
- "大部分人做 AI SaaS 站全是先搞站，那肯定难成功。"

**关键：** 开头不解释背景，直接用数字或痛点"钉住"读者。

### 正文技法

1. **铁律清单体**：标题数字承诺 → 每条规则一句话概括 → 口语化补充。天然适合截图传播
   - "Claude Code 入门：3 条铁律让你告别代码屎山"
   - 每条铁律都是**可立即执行的动作**，不是抽象建议

2. **填空公式**：把复杂概念压缩成公式
   - "__人在__情况下愿意花__钱来解决__问题"（需求定义）
   - "获取一个用户花多少钱 vs 一个用户能帮你赚多少钱"（商业模型）

3. **"不是 X，而是 Y" 认知重构**：
   - "不需要懂底层代码逻辑，但必须懂用户的需求逻辑"
   - "真正的门槛不是技术，是洞察需求的能力"
   - 这是他最常用的句式，每篇至少出现 2-3 次

4. **操作指令替代技术解释**：
   - 不说"Claude Code 基于 LLM，通过上下文窗口…"
   - 说"告诉它'不要急着写代码，先讨论方案'"
   - 用体验结果替代原理说明，对非技术读者极其友好

5. **粗粝口语**："屎山"、"无脑抄词"、"让用户骂"——打破技术内容的"体面感"，制造真实的信任感

### 结尾技法

**方法论品牌化收束**：不以金句结尾，而是把整篇方法论压缩成一个**可传播的标签**。

- "Idea To Business"
- "3 条铁律"
- "需求倒着做"

**特征：** 结尾是行动号召，不是哲学升华。"现在就去试"比"让我们拭目以待"强 100 倍。

### 禁忌

- 不讲技术原理（"NeRF 是什么"不重要，"8 秒出模型"才重要）
- 不做中立评价（必须有立场："这个方法我用了，有效 / 没效"）
- 不写没有数据支撑的观点

## 赛博禅心

> 小红书 + 公众号双平台创作者。内容方向：AI 工具实测、个人成长、数字生活方式。
> 以"禅心"为品牌调性——在快速变化的 AI 时代探讨慢下来、内省、有深度地使用工具。
> _（初始画像，基于有限观察。运行 /author-update 分析文章后完善。）_

### 核心风格：安静的洞察 + 第一人称真实感

与卡兹克的情绪爆发和花叔的方法论框架不同，赛博禅心走的是**安静、有质感的叙述路线**。不催促读者，不兜售焦虑，而是邀请读者进入一个相对安静的思考空间。

### 开头技法（初始观察）

1. **生活场景切入**：从日常的细节或感受开始，不是数据或事件
2. **提问引导**：开头用一个读者自己也在想的问题，制造认同感
3. **对比感受**：用"以前"和"现在"的体验对比，具体但不夸张

### 正文特征（初始观察）

1. **短段落节奏**：段落普遍较短（2-4 句），大量使用留白
2. **情绪词汇有温度但不极端**：不用"我靠""爽爆了"，而是"挺有意思的""感觉不一样了"
3. **工具实测带入个人使用情境**：不只说"好用"，而是说"在什么场景下我怎么用它，感受是什么"
4. **不下结论式观点**：倾向于"我自己的感受是"，给读者留思考空间

### 适配文章类型

- **creator-share**（工具分享）：最适配，分享亲测体验而非方法论
- **life-reflection**（生活反思）：自然匹配，适合年终/转折点内容
- **opinion**（观点）：可以，但需要注意不要太"碎"，要有一个清晰的中心判断

### 排版特征

- **公众号**：breathing-space 模板，短段落，少插图，以文字质感取胜
- **小红书**：封面文字克制，不用大字报式设计；图片偏生活感而非数据图

### 禁忌（初始观察）

- 不适合高密度信息轰炸型写法
- 不适合极端化标题（"100倍效率""颠覆认知"）
- 不用夸张情绪词

## 跨作者共性模式（写作时优先参考）

这些模式在所有标杆文章中都出现，是"好的 AI 领域文章"的底层结构：

### 1. 递进式升维

```
具体事件/个人体验（切入）
    → 现象描述（这件事意味着什么）
    → 框架/模型（为什么会这样）
    → 哲学/战略升维（这对我们意味着什么）
    → 一句话金句（重新定义一切）
```

每篇文章都是这个弧线。区别只是切入点和升维方向。

### 2. 具象优先原则

每个抽象概念必须配一个具象物：
- 不说"平台策略在收紧" → 说"先请你吃饭，再卖你保险"
- 不说"AI 能力在提升" → 说"以前需要一个团队，现在一个人 + AI 就够"
- 不说"这很重要" → 说"这是你这辈子真正活过的证据"

### 3. 信息密度铁律

每一段必须通过"删除测试"：如果删掉这一段文章不变差，这段就不该存在。

### 4. 数据锚定原则（来自刘小排）

每个核心论点必须有具体数字支撑，不能只有定性描述：
- 不说"成本大幅降低" → 说"成本降到原来的 1%"
- 不说"速度很快" → 说"8 秒完成"
- 不说"赚了很多钱" → 说"年营收超千万"

数字不是附属品，是内容的骨架。

### 5. 反 AI 感技巧

| 技巧 | 示例 |
|------|------|
| 短句碎句 | "别这样。" "不是这样的。" |
| 口语化 | "我靠"、"爽爆了" |
| 段落长短不一 | 有的段只有一句话，有的段 5-6 句 |
| 情绪突变 | 从冷静分析突然切到感叹 |
| 第二人称对话 | "你可能觉得…但其实…" |
| 不完整句 | 故意省略主语或宾语 |

### 5. 金句设计公式

标杆文章的收尾金句通常使用以下结构之一：

- **对仗反转**："X 最怕的，是 Y"（围墙花园最怕没有墙的花园）
- **重新定义**："X 不是 A。是 B。"（女娲造的不是人。是镜子。）
- **哲学升华**："X，可能是人类最 Y 的 Z"（跟概率较劲是人类最古老的浪漫）
- **存在主义锤**："那些你觉得太 A 了的东西，才是你真正 B 过的证据"
