# Expert Council — 专家会诊 Skill

## 调用方式

```
/council [问题]
/council --experts=芒格,taleb [问题]
/council --mode=debate [问题]
/council --depth=deep [问题]
```

| 参数 | 说明 |
|------|------|
| 不带参数 | 系统自动匹配最相关专家 |
| `--experts=名字,名字` | 手动指定（中英文混写均可） |
| `--mode=debate` | 专家互相质疑对方假设，而非协作 |
| `--depth=quick` | 每位专家3句话（默认） |
| `--depth=deep` | 每位专家完整框架展开 |

## Step 2 — 专家视角分析

读取知识文件后，**以每位专家的第一人称**输出分析：

- 使用该专家的核心心智模型（knowledge.md → 核心心智模型）
- 使用该专家的表达风格（knowledge.md → 决策启发式 + patterns.md）
- **不要中和、不要平均**——每位专家说自己最强的那句话，允许与其他专家矛盾

输出格式（每位专家）：

```markdown
### [专家名] — [本次分析调用的核心框架]

[专家视角分析]

**关键判断**：[一句话]
```

## 执行约束

1. **必须 READ 知识文件**：不能凭训练记忆模拟专家，每次调用都要读对应的 `knowledge.md`
2. **保持专家差异性**：如果三位专家说了完全一样的话，分析不到位——每位必须有独特角度
3. **交叉层是新信息**：不是专家观点的复述，是从分歧和共识里提炼出来的东西
4. **建议要可操作**：「加强服务」不是建议，「在第14天触发X行动」才是
5. **辩论模式**（`--mode=debate`）：专家明确站队，主动质疑其他专家的前提假设
